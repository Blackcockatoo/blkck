# Publication Design
## Frankston 2035 — Style Guide

*This guide governs all printed and digital publication outputs.*
*It does not govern the website. It does not govern social media.*
*It governs documents designed to be read as a coherent whole.*

---

## The Publication This Guide Is For

A beautifully made, physically printed document — and its digital equivalent — that presents Frankston 2035 to the world.

Not a brochure. Not a report. A *publication*.

Something a Council officer, a teacher, an artist, and a community member would each pick up and keep.

Something that feels like it was made with the same care as the work it describes.

---

## 1 — Typography

### Typeface System

**Primary — Display (headings, chapter titles, pull quotes):**
`Canela` — or substitute: `Freight Display` / `Tiempos Headline`
*Use: Elegant, humanist serif with warmth. Not cold institutional.*

**Secondary — Body (running text, captions, footnotes):**
`Neue Haas Grotesk` — or substitute: `GT America` / `Inter` (if open-source required)
*Use: Clean grotesque that reads well at small sizes. Never feels corporate.*

**Tertiary — Data / Tables / Labels:**
`IBM Plex Mono`
*Use: Tabular data, technical specifications, registers.*

### Type Scale

| Element | Size | Weight | Leading |
|---|---|---|---|
| Chapter title | 48pt | Light (300) | 52pt |
| Section heading | 24pt | Regular (400) | 30pt |
| Subheading | 16pt | Medium (500) | 22pt |
| Body text | 10pt | Regular (400) | 16pt |
| Caption | 8pt | Regular (400) | 12pt |
| Data / label | 8pt | Regular (400) | 12pt, tabular |
| Pull quote | 20pt | Light (300) | 28pt |

### Typography Rules

- Never justify body text. Ragged right only.
- Never use bold in body text except for the first instance of a key term.
- Never use italic for emphasis. Italic is for titles and foreign terms only.
- Minimum body text size: 9pt. No smaller.
- Pull quotes are never longer than two sentences.

## 2 — Colour System

### Primary Palette

| Name | Hex | Use |
|---|---|---|
| Deep Ink | `#1A1A1A` | Primary text, headlines |
| Warm White | `#F8F6F1` | Background, page |
| Frankston Blue | `#1B4F8A` | Primary accent, links, keylines |
| Coastal Sand | `#D4C4A0` | Secondary accent, dividers, warmth |

### Secondary Palette (Functional)

| Name | Hex | Use |
|---|---|---|
| Approval Green | `#3A7D44` | Permitted / yes / approved |
| Caution Amber | `#C97D1A` | Review required / uncertain |
| Stop Red | `#B83232` | Not permitted / rejected |
| Data Grey | `#6B6B6B` | Secondary text, labels, footnotes |

### Colour Rules

- Never use more than two palette colours on a single page (excluding text and background)
- Frankston Blue is used sparingly — it is an accent, not a wallpaper
- No gradients
- No transparency effects in print-facing documents
- Coastal Sand works as a block background for callout boxes and section dividers

## 3 — Grid

### Publication Grid

Format: A4 portrait (210mm × 297mm)
Margins: 20mm all sides (outer margin: 25mm for binding)
Columns: 12-column grid
Column gutter: 4mm
Baseline grid: 4mm

### Column Allocation

| Element | Columns |
|---|---|
| Full-width text block | 10 of 12 (centred) |
| Two-column layout | 5 / 5 with 2-column gutter |
| Image full bleed | 12 of 12 |
| Caption | 4 of 12, right-aligned |
| Pull quote | 8 of 12 (centred) |
| Diagram | 10 of 12 (centred) |
| Table | 10 of 12 |
| Sidebar | 3 of 12, right column |

### Spacing Rules

- Section break: 24mm vertical space
- Paragraph spacing: 1 baseline unit (4mm)
- Image to caption: 4mm
- Diagram to body text: 12mm above, 8mm below
- No widows or orphans (minimum 3 words on final line)

## 4 — Photography Direction

### What to Photograph

- People doing things, not posing
- The moment before and after the decisive moment — not the decisive moment itself
- Hands working
- Eyes reading
- Children concentrating
- Artists looking at their work
- Walls, before and during, not just after
- Empty library tables and full ones equally
- Frankston as it is, not as we wish it was

### What Not to Photograph

- Posed smiles
- People holding documents and looking at the camera
- "Diverse group" stock imagery framing
- Buildings without people
- Technology screens close-up (unless the screen content is the subject)

### Technical Direction

- Colour temperature: warm, natural light preferred
- Style: documentary, not editorial. Real over polished.
- No heavy post-processing. No HDR. No clarity slider abuse.
- Portraits: natural light, shallow depth of field, subject in their environment
- Minimum resolution for print: 300dpi at print size

### Consent

- All photographed participants sign a consent form
- Children: parent/guardian consent required before any photograph is taken
- No photograph of a child is published without verified consent on file

## 5 — Illustration Direction

### Style

- Line-based. Minimal fill. High contrast on white.
- Black ink weight varies to suggest depth without shadow or shading
- Human figures are stylised but warm — not corporate iconography
- Architecture and landscape are gestural — suggestion, not technical drawing
- No clip art. No stock illustration. Commissioned only.

### Use Cases

- Chapter openers: full-page illustration (abstract / atmospheric)
- Concept diagrams: clean line illustration
- Process breakdowns: line + minimal colour fill

### What Not to Illustrate

- Anything involving First Nations cultural imagery, symbols, or figures — without explicit community permission
- Anything that claims to depict an event or person that has not been photographically documented

## 6 — Icon Language

### System

Simple, consistent, monoline icons.
Stroke weight: 1.5pt at 24×24px
Corners: slightly rounded (2px radius)
Style: outline only, no filled icons

### Icon Set (Core)

| Icon | Represents |
|---|---|
| Location pin | Site, place, address |
| Person | Participant, role |
| Arrow (right) | Process, sequence, next step |
| Checkmark | Approved, completed |
| Cross | Rejected, not permitted |
| Clock | Timeline, duration |
| Leaf | First Nations awareness flag |
| Lock | Permission required, stage gate |
| Book | Library, resource, document |
| Brush | Artist, artwork, creation |
| Gear | Technology, tool |
| Group | Community, partnership |

### Rules

- Icons never appear alone — always paired with a label
- Icon size in body text: 16×16px inline
- Icon size in diagrams: 24×24px
- Never resize icons by stretching — only scale proportionally

## 7 — Diagram Style

*(See also: Visual Language document, chunk 07)*

All diagrams follow the rules in Chunk 07.

Additional publication-specific rules:

- Diagrams appear inside a 1pt hairline border
- Background: `#F8F6F1` (Warm White) — never white-on-white with page background
- Diagram title appears above, left-aligned, in body Medium
- Source / date appears below right, in 8pt Data Grey
- Minimum width: 80mm. Maximum: full text block width.
